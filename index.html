The Basics of The Web and HTML

The web is a collection of HTML documents. HTML stands for Hypertext Markup Language. HTML is the main type of document on the web. The major pieces to the web start with us on our browser. The browser makes HTTP requests to the internet that goes to the server and the server sends back the web page you wish to browse through the internet.

HTML is made up of:

text content: what you see

markup: what it looks like

references to other documents: i.e..images and videos

links to other pages: websites, compose email

HTML Tags:

b = bold

em = emphasis/italics

HTML Attributes:

href: the value of an anchor tag that links the media to the html document.

src: the value of an image tag that links the image to the html document.

alt: the value of the alternate text displayed in the image tag if the image won't load.

Image tags are called void tags since they don't have any content which is why they don;t require a closing tag.

Whitespace:

br = allows you to create a new line of text in html.

p = allows you to create a paragraph of content in html.

Inline and Block Elements

A, span, br, img, and strong are all inline elements because they don't generate a box around the content like a block elements does when you use p, div, or form.


Creating a Structured Document with HTML

HTML is the standard markup language used to create webpages.

CSS stands for Cascading Style Sheets. CSS is the language for how the page looks and is styled.

DOM stands for Document Object Model. DOM is a cross-platform and language independent convention for representing and interacting with the elements in HTML. DOM gives the elements in HTML the tree like structure.


Adding Style to Structure with CSS

More About CSS

CSS are rules for an HTML document to abide by that changes the format and style of all the elements in the page.

Selectors in the CSS are how you can change the rules to a specific element in a webpage.

A declaration is a line of code that controls the "background-color"(property) and the color "black"(value of that property).

Adding a class selector allows you to be more specific when dealing with multiple elements that are the same and want to style them separately.

To add a CSS style page to your html document you have to reference it in the head of your webpage.

Avoiding Repitition

The awesome advantage to CSS with HTML is eliminating repitition. No matter how often an element appears throughout your website. If the CSS declares a rule to that specific element, it will look the same throughout all the html pages that link to the CSS page. Making the programmer's life easier.

Input, Function and Output

What is a Function?

A function is something that takes input, does something to that input, then produces as output.

It is important to know what makes and uses a function to understand the code. Starting the line of code with def and giving specific parameters are what makes the function. Then the line of code that uses(calls) the function replaces the values that are using it.

Avoiding Repitition

Repitition is greatly decreased since once you have defined a function it can be reused over and over in the program. However, make sure you never forget your return statement. Forgetting this will cause the code not to know what it should produce and the print command will return with none.

Debugging

Debugging Strategy

Having a strategy when it comes to debugging is not only important but the only way you can keep frustration out of the picture when it comes up. You have to accept that errors happen and have a routine to fix them quick.

Step 1: Examine error messages when programs crash.

The last line of tracebacks will tell you where the error occurred. Read backwards till you find it.

Step 2: Work from example code.

If your proof read code doesn't work, comment that section out and do a step-by-step test to the code till it does what you want.

Step 3: Make sure examples work.

Always check the example code to see if it works in your system since that might not always be correct.

Step 4: Print intermediate results.

If your code doesn't crash and you don't get what you expected, add print statements to your program to find the misbehaving code.

Step 5: Keep and compare old versions.

Always find a good checkpoint to save your code elsewhere before attempting more so you can go back to it if you have too many errors.
